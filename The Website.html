The World's Most Medicore Game

by: Corbin Baly

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I thought it was pretty unique compared to what everybody else was doing. Also because I kinda ripped off the idea off another game I saw which was really similar to this one. I think I executed it pretty well in comparison to the original, and I had a few other ideas but I realized it would have been much harder. I wanted to make a kinda map game but it would have taken a lot of ai development, which I don't really have the skills to do right now.

2. What I Learned

I learned from user testing that the game was a little boring, so I decided to add little pieces of dialogue between the levels to give it a bit more personality. After I finished I kind of wanted to maybe even expand on that a bit more if I have time. The feedback was pretty helpful, also on making sure the levels are actually completable.

3. Architecture of My Game

There's a bunch of parts of the game, but they all work the same way with different variations on them. Like all the objects work the same way, they're drawn and have borders set around them by the bordercollisions class. Then they're used in the objectmanager to be managed in the correct positions, which will be decided in the gamepanel class. Which is the class that extends the JPanel class and is used as a panel in the runner.

4. Technical Implementation

Oh boy the technical implementation was pretty fun. My favorite aspect of the game was probably the changing of the size of the actual frame. It took way longer than it should have to be done, but in the end I found out that you can just set the preferred size of the frame, then just pack it. I think that this gives the game a lot more maneuverability to its levels, and makes it just feel much more unique.

5. Most Challenging Part

The most difficult part of the project was probably the obstacles, and making the rotate. To be honest I'm still confused at how it works, and I had to try a bunch of different times before I had some help. Some of my other ideas were to have the circles move in a square fashion, or even to move in an oval shape which I could have done much easier. But, I was determined to make sure that the end result was a circular movement.

6. Future Fixes and Updates

Maybe a few changes to the obstacles where they could rotate different directions, and change a few sizes to the last level so it could be a bit easier because from my experience, it's really hard.
